# Расчёт стоимости Claude API для автомобильной RE-платформы

## 1. Распределение вызовов по моделям

| Операция | Haiku | Sonnet | Opus | Обоснование |
|----------|-------|--------|------|-------------|
| APK перевод (извлечение строк) | 70% | 30% | 0% | Извлечение — тривиальная задача, перевод — Sonnet |
| Bug investigation | 0% | 85% | 15% | Сложный анализ logcat требует Sonnet, edge-cases — Opus |
| Voice setup (Level 3) | 10% | 40% | 50% | Мульти-агентная оркестрация — преимущественно Opus |
| Protection (Level 4) | 0% | 20% | 80% | Генерация обфускации требует максимального reasoning |
| Общие вопросы | 40% | 55% | 5% | Простые вопросы — Haiku, остальное — Sonnet |
| RAG / Knowledge base | 60% | 35% | 5% | Классификация и поиск — Haiku, синтез — Sonnet |

**Итого по платформе (взвешенно): ~35% Haiku, 42% Sonnet, 23% Opus**

## 6. Сводная таблица оптимизации

| Стратегия | Экономия | Комментарий |
|-----------|----------|-------------|
| Prompt caching | 30–40% | Системные промпты кэшируются между вызовами |
| Haiku для классификации | 15–20% | Извлечение строк, RAG-роутинг, простые ответы |
| Батчинг APK-переводов | 10–15% | Группировка строк из нескольких APK в 1 запрос |
| Локальный кэш ответов | 5–10% | Одинаковые строки между моделями авто |
| **Суммарная экономия** | **~45–55%** | Относительно наивного использования Opus на всё |

## 7. Ключевые выводы

1. **$15 за полную адаптацию 1 автомобиля** — API-косты пренебрежимо малы относительно трудозатрат
2. **~$2/день на воркера** — это ~$45/мес, что дешевле любого SaaS-инструмента
3. **Opus — главный драйвер расходов**: 23% вызовов дают ~70% стоимости. Каждый Opus-вызов стоит в 5–50× дороже Haiku
4. **Масштабирование линейное**: 20 воркеров ≈ $1,000–1,200/мес — это выполнимо
5. **Без оптимизации** (всё на Opus, без кэша): расходы вырастут до $3,500–5,000/мес на 20 воркеров. Оптимизация критична.

> **Итог: $50–60/мес на воркера при грамотной маршрутизации моделей. Сравните с зарплатой переводчика или RE-инженера — ROI очевиден.**
