01
Проблема

Адаптация китайских авто
для других рынков — сложно и долго

Каждый автомобиль — это уникальная задача: десятки APK, закрытые системы, специфические платформы. Работа требует глубокой экспертизы, которую сложно масштабировать.
⏱

Долгие сроки

Адаптация одного автомобиля занимает недели ручной работы. Перевод 20+ APK, патчинг сервисов, тестирование — всё вручную.

🔒

Зависимость от экспертов

Знания в головах 2-3 человек. Нанять нового — месяцы обучения. Потерять ключевого — потерять проект.

🔁

Потеря знаний

Каждая машина решается с нуля. Опыт по Deepal S07 не переносится на L07, хотя платформы похожи на 80%.

Решение

ИИ-агенты, которые помнят,
учатся и масштабируются

Платформа, где каждый проект делает систему умнее, а каждый новый сотрудник — сразу продуктивен.
🤖

ИИ-скиллы для каждой задачи

Специализированные ИИ-агенты для каждого этапа: взлом, перевод, виджеты, голосовое управление, защита. Работник просто описывает задачу.

🧠

Накопление знаний

Каждая находка сохраняется. Решения по одной машине автоматически предлагаются для похожей. База знаний растёт с каждым проектом.

🚀

Система ведёт работника пошагово. Не нужно знать smali — достаточно описать проблему. ИИ подскажет что и как делать.

ADB — мост к устройству

Скриншоты, установка APK, логи, запуск приложений, перезапуск сервисов — всё через ИИ. Работник не пишет adb команды.

Scrapy — разведка серверов

Сканирование API эндпоинтов, сбор серверных строк, проверка обновлений прошивок, анализ облачных конфигов авто.

Автоматический цикл

Нашёл баг → логи с устройства → анализ → исправление → сборка → заливка на устройство → проверка. Весь цикл в одном диалоге.

Фото бага = исправление

Работник описывает проблему. ADB тянет скриншот + логи. ИИ анализирует. Scrapy проверяет сервер. Фиксится и заливается обратно.

05
Архитектура скиллов

4 уровня сложности: от промпта
до мульти-агентной системы

Чем сложнее задача или ценнее реализация — тем глубже логика спрятана на сервере.
Уровень 1
Простой скилл
Только промпт-инструкция. Claude Code выполняет всё локально. Простые задачи с известным паттерном.
Промпт → Claude Code → Результат
split-screen patch-network systemui-adapt
✓ Работник видит промпт ✓ Работник видит код
Уровень 2
Скилл + MCP
Промпт + инструменты на сервере через MCP. Работник видит названия инструментов, но не их реализацию.
Промпт → Claude Code → MCP Сервер → apktool / jadx / подпись
apk-translate apk-rebuild adb-finder widget-builder
✓ Видит названия инструментов ✗ Не видит код инструментов
Уровень 3
Мульти-агент
Оркестратор управляет несколькими суб-агентами. Каждый со своим контекстом и MCP-инструментами. Сложные многошаговые задачи.
Оркестратор → Агент анализа + Агент патча + MCP сервер
voice-setup root-helper vuln-scanner release-build
✓ Видит результаты ✗ Не видит логику оркестрации ✗ Не видит суб-агентов
Уровень 4
Скрытая реализация
Вся логика на сервере. Работник не знает как это работает. Максимальная защита IP. Критичные проприетарные решения.
Вход: файл → ███ СЕРВЕР ███ → Выход: защищённый файл
trojan-embed resource-encrypt integrity-inject server-activation
✓ Видит только вход/выход ✗ Не видит НИЧЕГО внутри

👥

Почему мульти-агент?

Контекст LLM ограничен. Один агент не может держать в голове разрешения + API + патчи + базу знаний. Разделяем — каждый эксперт в своём.

🔌

Почему MCP?

Серверные инструменты запускают тяжёлые операции (декомпиляция, патч, сборка) без нагрузки на машину работника. И скрывают реализацию.

🔒

Почему скрытая реализация?

Алгоритмы защиты (трояны, шифрование, активация) — главный IP компании. Даже сотрудник не должен знать как они работают.

🌱

Рост сложности

Новый скилл начинается как Уровень 1 (промпт). По мере развития добавляются MCP, суб-агенты, скрытая логика. Эволюционный рост.
